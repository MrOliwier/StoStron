Bibliografia

Arch Linux to niezależnie rozwijana, zoptymalizowana pod kątem architektury x86_64 dystrybucja Linuksa skierowana do kompetentnych użytkowników Linuksa. Wykorzystuje „pacman”, swojego domowego menedżera pakietów, aby dostarczać aktualizacje najnowszych aplikacji z pełnym śledzeniem zależności. Działając w systemie wydań kroczących, Arch można zainstalować z obrazu CD lub za pośrednictwem serwera FTP. Instalacja domyślna zapewnia solidną podstawę, która umożliwia użytkownikom tworzenie instalacji niestandardowych. Ponadto Arch Build System (ABS) umożliwia łatwe tworzenie nowych pakietów, modyfikowanie konfiguracji pakietów podstawowych i udostępnianie tych pakietów innym użytkownikom za pośrednictwem repozytorium użytkowników Arch Linux. Arch Linux to niezależnie rozwijana dystrybucja GNU/Linux ogólnego przeznaczenia x86-64, która stara się dostarczać najnowsze stabilne wersje większości programów zgodnie z modelem stopniowego wydawania. Domyślna instalacja to minimalny system podstawowy, skonfigurowany przez użytkownika tak, aby dodawać tylko to, co jest celowo wymagane Arch Linux definiuje prostotę jako pozbawioną zbędnych dodatków i modyfikacji. Dostarcza oprogramowanie w postaci wydanej przez pierwotnych programistów (upstream) z minimalnymi zmianami specyficznymi dla dystrybucji (downstream): unika się łatek nieakceptowanych przez upstream, a łatki Archa downstream składają się prawie wyłącznie z przeniesionych poprawek błędów, które są przestarzałe w następnej wersji projektu. W podobny sposób Arch dostarcza pliki konfiguracyjne dostarczone przez upstream ze zmianami ograniczonymi do problemów specyficznych dla dystrybucji, takich jak dostosowanie ścieżek plików systemowych. Nie dodaje funkcji automatyzacji, takich jak włączenie usługi tylko dlatego, że pakiet został zainstalowany. Pakiety są dzielone tylko wtedy, gdy istnieją przekonujące korzyści, takie jak oszczędność miejsca na dysku w szczególnie poważnych przypadkach marnotrawstwa. Narzędzia konfiguracyjne GUI nie są oficjalnie dostarczane, co zachęca użytkowników do wykonywania większości konfiguracji systemu z poziomu powłoki i edytora tekstu.

By: Arkadiusz Marzewski, Oliwier Kubka
